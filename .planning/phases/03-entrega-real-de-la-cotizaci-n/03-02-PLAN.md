<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → EmailJS API | First outbound network call in the app; carries user-entered contact fields + quote summary |
| user input → email body | name/email/phone/message free-text crosses into the email template params |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | EmailJS public key exposed client-side (src/emailjs.config.js) | accept | EmailJS public key is non-secret by design (CONTEXT D-04); it only authorizes browser-side template sends. Service/template IDs scope sends to one template — no broader abuse surface. Document as accepted. |
| T-03-02 | Tampering / Injection | user-entered name/email/phone/message placed into email params (sendQuote) | mitigate | Trim and length-cap each field before building params (e.g. cap message length, single-line for name/email/phone). Validate email/phone format client-side (UI-SPEC validation contract). No HTML is constructed from these fields in-app (EmailJS template owns rendering), so no in-app XSS sink. |
| T-03-03 | Denial of Service / abuse | unauthenticated client send endpoint (EmailJS) | accept | Low-value target, no backend to protect; EmailJS dashboard rate-limits/quotas are the control. Out of code scope. |
| T-03-04 | Availability / graceful failure | missing or misconfigured credentials | mitigate | sendQuote throws `EMAILJS_NOT_CONFIGURED` when any of the three creds is empty; the modal catches it and shows the "El envío no está disponible… descargá el presupuesto en PDF" message instead of crashing (degrade to the working Plan-01 PDF path). |
| T-03-SC | Tampering | npm install of @emailjs/browser, html2pdf.js | mitigate | See Package Legitimacy Gate task below — verify both packages on npmjs.com before/at install; block on [ASSUMED]/[SUS]. |

Block only on high-severity: none here rise to blocking beyond the package-legitimacy checkpoint.
</threat_model>
