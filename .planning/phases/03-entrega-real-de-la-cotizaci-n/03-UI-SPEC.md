# Phase 3 — UI Design Contract

> Visual and interaction contract for **Entrega real de la cotización**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** This phase adds ONE new UI surface — a **contact-form modal** triggered by "Solicitar" — plus label/behavior changes to the existing cotizador action bar. Per CONTEXT.md (D-06, "Claude's Discretion") the modal **reuses the existing AR modal/overlay pattern** and the **ephemeral "copiado" feedback idiom** for visual coherence. **No visual redesign** of the rest of the app (locked constraint). All tokens below are extracted from the existing `styles/styles.css` — this contract codifies what already exists so the new modal matches it exactly.

## Spacing Scale

Project uses a near-8pt scale already present in `styles/styles.css` (modal padding 24px, hint padding 14×18, close button 38px, gaps 10px). Declared values for this phase (all multiples of 4; the existing 14/18/10 odd values are pre-existing and out of scope to change):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, validation-error icon/text gap |
| sm | 8px | Field label → input gap, button glyph gap |
| md | 16px | Default gap between form fields, input vertical padding |
| lg | 24px | Modal outer overlay padding (matches `.ar-modal` `padding:24px`), modal box internal padding |
| xl | 32px | Vertical breathing room between form block and status/footer area |

Exceptions (pre-existing in stylesheet, reused as-is, NOT introduced by this phase): close button 38px touch target, modal hint padding `14px 18px`, button gap `10px`. New form controls should round to the 4pt scale above.

## Typography

All roles map to existing `styles.css` conventions (Archivo, uppercase letter-spaced labels, mono numerics). The new modal must NOT introduce new sizes/weights beyond these. **Exactly two weights are permitted: 400 (regular) and 700 (bold).** Title dominance is achieved by size alone (20px vs 12px labels), not by a heavier weight.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display / modal title | 20px | 700 | 1.2 |
| Body / input text + status message | 14px | 400 | 1.5 |
| Label (field labels, eyebrow `.tag`) | 12px | 700 (uppercase, letter-spacing 0.16em) | 1.4 |
| Button | 12px | 700 (uppercase, letter-spacing 0.16em — matches `.btn`) | 1 |

Weights in use: **exactly two** — **400 (regular)** for body/inputs, **700 (bold)** for the modal title, labels, and buttons. The modal title is distinguished by its larger size (20px) rather than a third weight. Numeric total in the email summary (if shown) uses the `.mono` (JetBrains Mono) class already defined.

## Color

Dark theme. Values are the existing CSS custom properties from `:root` in `styles/styles.css` — reused verbatim, none introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` `#080807` / `--bg-2` `#0e0e0d` | App background; modal overlay base; modal box surface uses `--bg-2` (matches `.ar-modal__box`) |
| Secondary (30%) | `--panel` `#121211` / `--panel-2` `#181816` + `--line` `rgba(255,255,255,.1)` | Input field surfaces, field borders, section dividers, status-area background |
| Accent (10%) | `--red` `#e23a2b` (hover `--red-soft` `#f1543f`) | RESERVED list below |
| Destructive | not applicable | This phase has **no destructive actions** (no delete/irreversible flows — only download PDF and send email) |

**Accent reserved for:**
- The primary CTA button only — "Solicitar / Enviar cotización" (`.btn--red`, the single most important action on the bar/modal).
- The brand glyph "reno**.**" red dot inside the generated PDF document (pre-existing `.acc` class).
- Validation-error and send-error inline text/border on form fields (semantic re-use of `--red` for the error state — consistent with the app having no separate error color).

Accent is **never** applied to the secondary actions ("Descargar ficha técnica", "Descargar presupuesto" stay as neutral `.dlbtn`), field labels, or the success state. **Success state uses neutral ink (`--ink` on `--panel`), not green** — the palette has no success color and CONTEXT forbids new visual identity; success is conveyed by copy + a neutral check glyph, matching the existing ephemeral "¡Link copiado!" idiom.

## Copywriting Contract

Language: **Spanish (es)** — all user-facing copy. Button copy and validation regex are Claude's discretion per CONTEXT.md (D-08); the values below are the prescribed copy for this phase.

### Action bar (cotizador) — honest labels (QUOTE-03 / D-08)

| Element | Copy |
|---------|------|
| Secondary action — technical sheet | `↓ Descargar ficha técnica (PDF)` |
| Secondary action — quote | `↓ Descargar presupuesto (PDF)` |
| Primary CTA (opens modal) | `Solicitar cotización →` |

Rationale: documents are now real PDFs (D-01/D-02), so "(PDF)" is honest. "Solicitar cotización" now genuinely sends to the business (D-04), so the label no longer over-promises.

### Contact-form modal (QUOTE-02 / D-06, D-07)

| Element | Copy |
|---------|------|
| Modal title | `Solicitá tu cotización` |
| Modal subtitle | `Te enviamos esta configuración y un asesor te contacta.` |
| Field — nombre (required) | label `Nombre *` · placeholder `Tu nombre` |
| Field — email | label `Email` · placeholder `tu@email.com` |
| Field — teléfono | label `Teléfono` · placeholder `+54 9 ...` |
| Field — mensaje (optional) | label `Mensaje (opcional)` · placeholder `Contanos algo más…` |
| Contact-method hint | `Dejanos al menos un email o teléfono para poder responderte.` |
| Submit button — idle | `Enviar cotización →` |
| Submit button — disabled hint | (button simply disabled when invalid; no separate copy) |

### Three runtime states (in-modal, D-07)

| State | Copy / behavior |
|-------|-----------------|
| **Enviando** (sending) | Submit button disabled, label becomes `Enviando…` with an inline spinner/indicator. Form inputs disabled. |
| **Éxito** (success) | Replace form with confirmation: heading `¡Cotización enviada!` + body `Te contactamos pronto. Revisá tu casilla.` Neutral ink + check glyph (no green). Modal stays open with a `Cerrar` button. |
| **Error** (error + retry) | Inline error block: `No se pudo enviar tu cotización.` + body `Revisá tu conexión e intentá de nuevo.` + retry button `Reintentar`. **Form data is preserved** (not cleared) so the user retries without re-typing. |
| Missing-credentials error (defensive) | If EmailJS credentials are absent/misconfigured, show the same Error block with body `El envío no está disponible en este momento. Probá descargar el presupuesto en PDF.` (graceful fallback to the working download path). |

**Control-label note:** `Cerrar` and `Reintentar` are intentionally single-word control labels — acceptable inside a modal where context (the surrounding heading/body copy) supplies the missing object. They are not orphaned verbs. If desired, the executor may use the slightly more explicit pair `Cerrar` / `Reintentar envío`; keeping the single words as-is is also approved.

### Validation contract (D-06, Claude's discretion on exact regex)

| Field | Rule |
|-------|------|
| nombre | Required; non-empty after trim. |
| email | Optional, but if present must match a permissive email regex (e.g. `^\S+@\S+\.\S+$`). |
| teléfono | Optional, but if present must contain ≥ 8 digits (strip spaces/`+`/`-`). |
| Cross-field | At least ONE of email/teléfono must be present AND well-formed. |
| Send enablement | Submit disabled until: nombre valid AND (valid email OR valid teléfono). Errors shown inline per field on blur/submit, using `--red` text/border. |

### Empty / not-applicable states

| Element | Copy |
|---------|------|
| Empty state | **Not applicable** — the modal always opens from a fully configured quote (there is no data-less entry point); the quote total/config always exists when "Solicitar" is reachable. |

## Interaction Contract (modal mechanics — reuse existing patterns)

| Aspect | Contract |
|--------|----------|
| Open trigger | Click on `Solicitar cotización →` in `.cotizar__actions` (`src/Studio3D.jsx:713`). |
| Overlay | Reuse `.ar-modal` overlay: `position:fixed; inset:0; z-index:60; background:rgba(0,0,0,.78); backdrop-filter:blur(6px); display:grid; place-items:center; padding:24px`. Click on overlay (outside box) closes the modal — same as AR modal. |
| Box | Reuse `.ar-modal__box` look: `background:var(--bg-2); border:1px solid var(--line-strong); border-radius:16px`. Size for a form, not a 3D viewer: `width:min(440px,94vw)`, height auto. New class (e.g. `.contact-modal__box`) may extend the same visual tokens rather than literally reusing the AR sizing. |
| Close | Reuse `.ar-modal__close` button (`✕`, 38×38, top-right) and `e.stopPropagation()` on the box — identical to AR modal. The `✕` button MUST carry `aria-label="Cerrar"` (low-cost; the glyph alone has no accessible name). Closing in success/error state is always allowed. |
| Ephemeral feedback idiom | Transient confirmations follow the `copyLink`/`setCopied` + `setTimeout(…, 1800)` idiom (e.g. the ephemeral "¡Link copiado!" toast). The success state, however, is a **persistent** in-modal panel (not a 1.8s toast) because the user needs to read it (D-07). |
| Focus | On open, focus the `Nombre` field. Esc closes (parity with overlay click). (Full a11y/keyboard hardening is Phase 5 scope — do not over-build here, but Esc + autofocus + the close-button `aria-label` are low-cost and match AR-modal expectations.) |
| State machine | Local `useState` in StudioScreen: `idle → sending → (success | error)`; `error → sending` on Reintentar; closing resets to `idle`. Inputs are controlled state; preserved across error→retry. |
